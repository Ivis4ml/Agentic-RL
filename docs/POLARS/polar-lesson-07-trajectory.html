Trajectory 构建与评估

能访问 CDN 时切换为交互版。本课讲清 Polar 如何把 agent 一次 rollout 期间产生的多次独立 LLM 调用，重建为 RL 训练器可直接消费的 token 级轨迹（Trajectory）：per_request 与 prefix_merging 两种构建策略、prefix_merging 的分组与合并算法、loss_mask 与 logprob 的填充规则，以及 swebench_harness / test_on_output / session_completed 三类评估器如何把奖励写回轨迹。
